[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
(pause)
Let me walk you through how NutriPulse processes incoming user requests behind the scenes.
First, every request enters our Security Checkpoint node to perform instant PII scrubbing and prompt injection checks.
If passed, execution moves to Orchestrator Prep, which initializes shared state data before kicking off parallel branch execution.
In parallel, our Meal Planner agent generates structured daily calorie and macro breakdowns, while our Wellness Coach agent formulates custom hydration and workout targets.
Both specialized agents connect to our local MCP Server via stdio transport to query live nutrition algorithms and recipe databases.
Their outputs join at Join Plans and flow into a Human Approval node, pausing execution so the user can review the proposed plan before our final summary node synthesizes the complete report.
